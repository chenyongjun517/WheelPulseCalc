# 项目结构说明

## 目录划分

```
WheelPulseCalc_MiniApp/
├── apk_build/          # Android 专用目录（保持不变）
│   ├── www/           # H5 源码
│   ├── android/       # Android 原生项目
│   └── ...
│
├── ios_build/         # iOS 专用目录（GitHub Actions 自动创建）
│   ├── www/          # 从 apk_build 复制的源码
│   ├── ios/          # iOS 原生项目
│   └── ...
│
├── pages/             # 微信小程序源码
│
├── .github/workflows/ # GitHub Actions 配置
│   └── build-ios.yml
│
└── init_ios.bat       # 本地初始化 iOS 目录脚本
```

## 使用规则

### Android 开发
- **只修改** `apk_build/` 目录下的文件
- 不要动 `ios_build/` 目录
- 构建命令：`build_apk.bat`

### iOS 开发
- **只修改** `ios_build/` 目录下的文件
- 不要动 `apk_build/` 目录
- GitHub Actions 会自动创建 `ios_build` 并构建

### 微信小程序
- 修改 `pages/` 目录
- 与 APK/iOS 无关

## 代码同步策略

如果需要在 Android 和 iOS 之间同步源码修改：

1. 先在 `apk_build/www/` 修改并测试
2. 手动复制到 `ios_build/www/`
3. 或者重新运行 `init_ios.bat` 重新创建 iOS 目录

## GitHub Actions

推送代码到 GitHub 后，会自动：
1. 创建 `ios_build` 目录
2. 复制 `apk_build/www` 源码
3. 添加 iOS 平台并构建
4. 生成 iOS Simulator 安装包（.zip）
5. 可在 Actions 页面下载构建产物

**注意：** iOS 真机安装需要 Apple 开发者证书，GitHub Actions 默认只构建 Simulator 版本用于测试。
